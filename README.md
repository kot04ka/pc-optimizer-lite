# PC Optimizer Lite

Лёгкое Python-приложение для пассивного мониторинга Windows/Linux/macOS через `psutil`.
Основной GUI сделан на PySide6: тёмная тема, карточки метрик, живые графики и tray-режим.

## Что умеет

- CPU: общая загрузка и загрузка по ядрам.
- RAM: занято, доступно, процент использования.
- Диски: место по разделам и приблизительная скорость чтения/записи.
- Процессы: CPU/RAM, RSS, priority, путь к exe.
- Современный UI: тёмная/светлая тема, QSS, карточки, цветовая индикация нормы/нагрузки/критики.
- Лёгкий real-time график CPU/RAM без matplotlib/pyqtgraph: `deque(maxlen=60)`, обновление только в cadence монитора, EMA-масштаб Y.
- Главная кнопка "Оптимизировать": one-click цикл scan → classify → RAM/CPU relief → sleep/close → cleanup → report.
- Оптимизация CPU: настраиваемое понижение priority + опциональное временное CPU affinity-ограничение с автовозвратом.
- Кнопка "Очистить RAM": лёгкий режим EmptyWorkingSet для безопасных неактивных процессов; глубокий режим дополнительно пытается очистить системные списки памяти и требует прав администратора.
- Вкладка "Активность": события, история закрытых процессов, кнопка "Открыть снова".
- Режим сна: оконные приложения получают мягкий priority sleep, deep suspend остаётся только для headless-safe кандидатов, ручное и автоматическое пробуждение.
- Low-power режим для слабых ПК: более редкий polling, упрощённый график и опциональное временное отключение части визуальных эффектов Windows с восстановлением при выходе.
- Умное закрытие фоновых процессов: выключено/спрашивать/авто, с whitelist и проверкой активного приложения.
- Уведомления при превышении порогов CPU/RAM с антиспам-кулдауном.
- Безопасное “облегчение”: понижение priority/affinity у тяжёлых процессов вне whitelist без kill().
- Whitelist системных процессов и пользовательских исключений по имени или пути.
- Очистка только известных temp/cache-директорий и только после подтверждения, с отчётом по категориям.
- Логирование действий в `%APPDATA%\PC Optimizer Lite\pc_optimizer_lite.log`.

## Безопасность

По умолчанию включён режим "только наблюдения": авто-действия запрещены.
Системные и whitelisted процессы не трогаются ни закрытием, ни сном, ни priority-изменениями.
Авто-закрытие отдельно настраивается: выключено, спрашивать подтверждение или разрешить авто-закрытие.
One-click оптимизация перед первым запуском показывает отдельное consent-окно с чекбоксом "Понимаю и согласен".
Внутри one-click цикла нет дополнительных подтверждений, но whitelist, активное окно, сеть/медиа-признаки и сомнительные процессы всё равно защищены.
Очистка RAM не закрывает приложения и не применяется к whitelist, активному foreground-приложению, медиа/аудио-процессам и процессам с активной сетью.
CPU-оптимизация работает внутри общего snapshot-цикла: whitelist не трогается, процессы не завершаются, а realtime-признаки блокируют affinity-урезание.
Удаление файлов выполняется только после подтверждения пользователя и только в заранее известных temp/cache-папках.
История закрытых процессов хранится в `%APPDATA%\PC Optimizer Lite\history.json`.

## Запуск

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

```

Если `plyer` недоступен, уведомления будут просто записываться в лог.

## Упаковка

```powershell
.\build.bat
```

Скрипт сборки:

- генерирует иконку `assets\pc_optimizer_lite.ico`;
- собирает onedir-папку PyInstaller, чтобы приложение не распаковывалось в `%TEMP%\_MEI` при запуске;
- собирает `PC-Optimizer-Lite-windows-x64.zip` как portable/diagnostic onedir-архив;
- включает hidden imports для PySide GUI, уведомлений, pywin32/WMI и внутренних модулей `autostart`, `optimize_action`, `ram_cleaner`, `cpu_optimizer`, `cpu_throttler`, `sleep_manager`, `safety.activity_detector`, `history_manager`;
- собирает установщик через Inno Setup (`installer\PC Optimizer Lite.iss`), который ставит всю onedir-папку в `%LOCALAPPDATA%\Programs\PC Optimizer Lite`.

Для локальной сборки setup exe нужен установленный Inno Setup 6 (`iscc.exe`). GitHub Actions workflow устанавливает Inno Setup перед релизной сборкой автоматически.

## Автообновления через GitHub

Приложение по умолчанию проверяет GitHub Releases репозитория `kot04ka/pc-optimizer-lite`.
Для публикации новой версии:

- соберите onedir ZIP и установщик через `.\build.bat`;
- создайте GitHub Release с tag вида `v1.4.0` или выше текущей версии;
- прикрепите assets `PC-Optimizer-Lite-windows-x64.zip` и `PC-Optimizer-Lite-Setup.exe`;
- добавьте в описание релиза строки `PC-Optimizer-Lite-windows-x64.zip sha256: <хэш>` и `PC-Optimizer-Lite-Setup.exe sha256: <хэш>`.

Репозиторий публичный, поэтому токен не нужен: приложение читает GitHub Releases API без авторизации. В пользовательских настройках нет полей owner/repo/token; для пользователя остаётся одна кнопка "Проверить обновления", которая меняется на "Обновить до X.X", когда найден новый релиз.

Автообновление скачивает `PC-Optimizer-Lite-Setup.exe`, проверяет размер и sha256 до запуска, стартует установщик из `%TEMP%` в тихом режиме `/VERYSILENT /SUPPRESSMSGBOXES /NORESTART` и сразу закрывает приложение. Замену файлов выполняет Inno Setup с `CloseApplications=yes` и `RestartApplications=yes`; самописная замена/переименование папки приложения не используется.

Если установленная версия уже не запускается с ошибкой `_PYI_APPLICATION_HOME_DIR environment variable is not defined!` / `Failed to load Python DLL ... _MEI...\python311.dll/python312.dll` или зацикленно предлагает `1.3.8`, автообновление из этой версии невозможно. Нужно один раз вручную скачать `PC-Optimizer-Lite-Setup.exe` из GitHub Releases и установить поверх старой версии. После этого следующие автообновления будут идти через исправленный установщик. Если PyInstaller-ошибка осталась, удалите папки `_MEI*` из `%TEMP%` вручную и переустановите.

## Готовая сборка

Версия: `1.5.0`

Дата сборки: `2026-06-28`

Артефакты локальной сборки (`.\build.bat`) находятся в:

```text
installer_output\PC-Optimizer-Lite-windows-x64.zip

```

> **Важно:** SHA256-хэши, публикуемые на GitHub Releases, отличаются от хэшей локальной сборки — PyInstaller не даёт воспроизводимых бинарников. Авторитетные хэши для проверки скачанного файла всегда берутся из тела GitHub Release (туда их записывает CI-workflow).

Команда пересборки с нуля из корня проекта:

```powershell
.\build.bat
```

Установщик ставит приложение в пользовательскую папку `%LOCALAPPDATA%\Programs\PC Optimizer Lite`, создаёт ярлык в меню "Пуск", опционально создаёт ярлык на рабочем столе и может включить автозапуск через `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` с флагом `--tray`.

Неподписанный EXE может вызвать предупреждение Windows Defender SmartScreen. Приложение и установщик не отключают Windows Defender, антивирусы или другие средства защиты. Для публичного распространения лучше подписать файл кодовым сертификатом; для локального использования можно подтвердить запуск вручную.

- Сворачивание в трей включает энергосбережение: редкий мониторинг, редкое обновление процессов, без графика.
- При старте приложение понижает собственный priority до BELOW_NORMAL.

### 1.1.0

- Добавлена настраиваемая оптимизация CPU в общем цикле: priority + affinity без завершения процессов.
- CPU/RAM/sleep/periodic optimization используют единый snapshot цикла без повторного `psutil.process_iter()`.
- Добавлен автовозврат CPU priority/affinity и логирование изменений в ленту "Активность".
